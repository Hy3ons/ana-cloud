# Design

1. 담백한 최신 포털모양으로 디자인을 하십시오. 텍스트 버튼 기반의 브라우저를 만들어서 최대한 많은 기능들을 한 화면에 채울수 있도록 구현하세요.
2. 그렇다고 너무 올드한 느낌이 들면 안됩니다. 주 고객층은 20-30대의 컴퓨터공학도 생들입니다.
3. 파란색 계열의 Azure 스타일의 클라우드 플랫폼 (VM만 지원) 하는 사이트를 만들돼, 살짝 더 어두운 색감으로 느낌을 살립니다.

# Optimizing 
1. 최대한 필요하지 않은 import 나 파일을 제거하고 가볍고 가독성 좋게 합니다.
2. 오버엔지니어링을 하지 않도록 합니다. 엔터프라이즈 급이 아니기 때문에, 과도한 try catch를 지향하고, 에러 자체를 안나게 설계합니다. 단 단일 장애 지점이 없게 하여 시스템이 중지되는 경우는 절대로 없대록 합니다.
